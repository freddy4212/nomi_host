The Control Panel receives data via **TCP** using `nomi_host/observation_layer/modules/network/receiver.py`. This receiver expects a "Rich" JSON format (containing full skeleton data, confidence scores, 3D coordinates, and optional images).

The Simulator (`virtual_endpoint_simulator`) currently:
1.  Converts the data to **YOLO format** (stripping much of the "Rich" data).
2.  Sends it using `PacketSender`, which is currently implemented using **TCP** (`socket.create_connection`).

**Conclusion:**
- **Control Panel cannot handle the YOLO format:** It expects the Rich format with specific fields that are removed in the YOLO conversion.
- **Control Panel does not listen on UDP:** It listens on TCP.
- **Current Simulator behavior is conflicting:** It sends YOLO format (intended for external tools) over TCP (where Control Panel is listening), which breaks the Control Panel's visualization.

**Recommended Solution:**
Implement a "Dual Output" or "Split Mode" in the Simulator:
1.  **UDP Output (YOLO format):** Send the simplified YOLO JSON over UDP to the external receiver (e.g., YOLOv8 script).
2.  **TCP Output (Rich format):** Send the full Rich JSON over TCP to the Control Panel for visualization.

I will now modify the Simulator to support sending UDP packets and allow it to send both formats simultaneously if needed (or at least clarify the separation).

First, I will modify `nomi_host/virtual_endpoint_simulator/backend/app/services/sender_service.py` to support UDP.
Then, I will update `nomi_host/virtual_endpoint_simulator/backend/app/api.py` to allow specifying protocol (TCP/UDP) in the request, or automatically use UDP for YOLO format.
